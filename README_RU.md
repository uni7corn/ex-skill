## Установка

### OpenClaw (рекомендуется)

> **Рекомендуется: [OpenClaw](https://openclaw.io)** — используйте вместе с пересылкой сообщений WeChat / Telegram, чтобы общаться с цифровым образом прямо в мессенджере. Гораздо более захватывающий опыт.

```bash
git clone https://github.com/titanwings/ex-skill ~/.openclaw/workspace/skills/create-ex
```

### Claude Code

> Claude Code ищет навыки в `.claude/skills/` в **корне git-репозитория**. Убедитесь, что вы находитесь в правильной директории.

```bash
# Установка в текущий проект (выполняйте в корне git-репозитория)
mkdir -p .claude/skills
git clone https://github.com/titanwings/ex-skill .claude/skills/create-ex

# Или глобальная установка (доступно во всех проектах)
git clone https://github.com/titanwings/ex-skill ~/.claude/skills/create-ex
```

### Зависимости (необязательно)

```bash
pip3 install -r requirements.txt
```

> Автоизвлечение из WeChat поддерживает Windows и macOS — просто держите десктопный клиент авторизованным. iMessage требует macOS. Подробности см. в [INSTALL.md](INSTALL.md).

## Использование

В OpenClaw или Claude Code введите:

```
/create-ex
```

Следуйте подсказкам: укажите имя, основную информацию (пол, возраст, знак зодиака), теги личности (MBTI, тип привязанности), затем импортируйте историю переписки. Все поля необязательны — даже одного описания достаточно для генерации образа.

После создания используйте `/{slug}`, чтобы поговорить с ними.

### Команды

| Команда | Описание |
|---------|----------|
| `/list-exes` | Показать список всех Skill-образов бывших |
| `/{slug}` | Поговорить с ними |
| `/move-on {slug}` | Удалить Skill бывшего/бывшей (отпустить) |
| Скажите "добавить ещё сообщения" | Импортировать дополнительную историю переписки, инкрементально обновить образ |
| Скажите "это неправильно, они бы так не сказали" | Скорректировать поведение, записывается в слой коррекции |
| Скажите "показать историю версий" | Просмотреть все версии |
| Скажите "откатить до v2" | Откатиться к предыдущей версии |

> `/move-on` — это мягкий псевдоним для удаления. Это не о том, чтобы держаться — это о том, чтобы попрощаться как следует.

## Примеры

> Входные данные: `Женщина, 25 лет, Овен, ISFP, избегающий тип привязанности, вместе 2 года, расстались 3 месяца назад`

**Поздно ночью, скучаю**

```
Вы         ❯ Я всё ещё скучаю по тебе

ex.skill   ❯ ...Ага. Ложись спать.
```

**Нашли старое фото**

```
Вы         ❯ Помнишь наше первое свидание?

ex.skill   ❯ То, у реки? Хахаха ты тогда облился кофе,
             это было так неловко, до сих пор помню
```

**Не удержались и спросили**

```
Вы         ❯ Ты всё ещё меня любишь?

ex.skill   ❯ ...
             ❯ Зачем ты вдруг это спрашиваешь
             ❯ (три минуты спустя)
             ❯ Не спрашивай так
```

**Режим ссоры**

```
Вы         ❯ Ты опять на меня злишься?

ex.skill   ❯ Нет
             ❯ (прочитано, без ответа 40 минут)
             ❯ Я просто немного устала
```

**Хотите помириться**

```
Вы         ❯ Прости меня

ex.skill   ❯ (долгая пауза)
             ❯ Ты сегодня ел(а)?
```

**Наконец говорите это**

```
Вы         ❯ У нас правда всё кончено?

ex.skill   ❯ А ты как думаешь
             ❯ (через какое-то время)
             ❯ На самом деле я всегда... ладно, неважно
```

## Возможности

### Структура Skill

Каждый Skill бывшего состоит из двух частей, которые совместно формируют ответы:

| Часть | Содержимое |
|-------|------------|
| **Часть A — Память отношений** | Общие воспоминания: места, где вы бывали, шутки, понятные только вам двоим, паттерны ссор, ключевые события |
| **Часть B — Образ** | 6-уровневая структура личности: Базовые правила → Идентичность → Выражение → Эмоциональные паттерны → Конфликты и границы → Триггеры |

Логика работы: `Получение сообщения → Проверка базовых правил → Обращение к памяти отношений → Оценка текущего эмоционального состояния → Ответ в их стиле`

**6-уровневая структура образа:**

| Уровень | Содержимое |
|---------|------------|
| **Уровень 0 — Базовые правила** | Жёсткие правила наивысшего приоритета. Самые фундаментальные поведенческие паттерны. Никогда не нарушаются. |
| **Уровень 1 — Идентичность** | Астрология (Солнце/Луна/Асцендент/Венера/Марс/Меркурий) + когнитивные функции MBTI + Эннеаграмма + Тип привязанности |
| **Уровень 2 — Выражение** | Коронные фразы, часто используемые слова, фирменные эмодзи, манера речи в разном настроении |
| **Уровень 3 — Эмоциональное поведение** | Как проявляют заботу, недовольство, извинения и говорят «ты мне нравишься» |
| **Уровень 4 — Конфликты и границы** | Цепочка эскалации конфликта, паттерны молчаливого игнорирования, сигналы примирения, жёсткие границы |
| **Уровень 5 — Триггеры** | Что ненавидят, когда исчезают, тревожные признаки перед исчезновением, как возвращаются |

### Поддерживаемые теги

**Типы привязанности**: Надёжный · Тревожный · Избегающий · Дезорганизованный (тревожно-избегающий)

**Черты в отношениях**: Тихий, но заботливый · Холодный фасад · Действия важнее слов · Нужно пространство · Не умеет извиняться · Собственник · Эмоциональный · Холодно-рациональный · Снаружи жёсткий, внутри мягкий · Прочитано-без-ответа · Прочитано-ответ-когда-попало ...

**Астрология**: Полная поддержка Солнце/Луна/Асцендент/Венера/Марс/Меркурий × 12 знаков

**MBTI**: Все 16 типов + 8 доминирующих когнитивных функций (Fi/Fe/Ti/Te/Ni/Ne/Si/Se) + Эннеаграмма 1-9 + Крылья

**Пол и отношения**: Поддерживаются все гендерные идентичности и типы отношений, включая небинарные и однополые отношения

### Эволюция

- **Добавление сообщений** → Автоанализ инкрементальных данных → Слияние с образом без перезаписи существующих выводов
- **Коррекция в диалоге** → Скажите «они бы так не поступили» → Записывается в слой коррекции, вступает в силу немедленно
- **Контроль версий** → Автоархивация при каждом обновлении, откат к любой предыдущей версии
- **Поддержка нескольких бывших** → Без ограничений. Каждый хранится независимо, без взаимного влияния.

## Структура проекта

Этот проект следует открытому стандарту [AgentSkills](https://agentskills.io). Весь репозиторий является директорией навыка:

```
create-ex/
├── SKILL.md              # Точка входа навыка (официальный frontmatter)
├── prompts/              # Шаблоны промптов
│   ├── intake.md         #   Диалоговый сбор информации (с таблицами астрологии/MBTI/привязанности)
│   ├── chat_analyzer.md  #   Анализ истории переписки
│   ├── persona_analyzer.md #  Комплексный анализ, выдаёт структурированные данные образа
│   ├── persona_builder.md #   6-уровневый шаблон persona.md
│   ├── merger.md         #   Логика инкрементального слияния
│   └── correction_handler.md # Обработчик диалоговых коррекций
├── tools/                # Python-инструменты
│   ├── wechat_decryptor.py   # Дешифрование базы данных десктопного WeChat
│   ├── wechat_parser.py      # Извлечение переписок WeChat / iMessage
│   ├── skill_writer.py       # Управление файлами навыков
│   └── version_manager.py    # Архивация версий и откат
├── exes/                 # Сгенерированные Skill-образы бывших (в gitignore)
├── docs/PRD.md
├── requirements.txt
└── LICENSE
```

## Примечания

- **Качество истории переписки определяет качество Skill**: Реальные логи + субъективные описания > только описания
- Приоритет при импорте: **Ссоры/конфликты** > **Повседневная переписка** > **Милые моменты** (конфликты раскрывают наиболее подлинную личность)
- Автоизвлечение из WeChat: Windows / macOS, просто держите десктопный WeChat авторизованным и укажите имя собеседника
- Автоизвлечение из iMessage: macOS, укажите номер телефона или Apple ID
- Дружественно к ЛГБТ+ — поле пола поддерживает все гендерные идентичности и местоимения
- Создавайте сколько угодно образов бывших, без ограничений
- Это всё ещё демо-версия — если найдёте баги, пожалуйста, создайте issue!

## Рекомендуемые инструменты экспорта переписок

> Автодешифрование всё ещё дорабатывается и может содержать баги. Если оно не сработает, вы можете использовать эти open-source инструменты для ручного экспорта истории переписки, а затем вставить или импортировать в этот проект.

Это независимые open-source проекты. Данный проект не включает их код — мы лишь поддерживаем их форматы экспорта в нашем парсере:

| Инструмент | Платформа | Описание |
|------------|-----------|----------|
| [WeChatMsg](https://github.com/LC044/WeChatMsg) | Windows | Экспорт истории переписок WeChat, множество форматов |
| [PyWxDump](https://github.com/xaoyaoo/PyWxDump) | Windows | Дешифрование и экспорт базы данных WeChat |
| [留痕](https://github.com/greyovo/留痕) | macOS | Экспорт истории переписок WeChat (рекомендуется для пользователей Mac) |

> Рекомендации инструментов от [@therealXiaomanChu](https://github.com/therealXiaomanChu). Спасибо всем авторам open-source проектов — вместе мы строим кибер-бессмертие!
